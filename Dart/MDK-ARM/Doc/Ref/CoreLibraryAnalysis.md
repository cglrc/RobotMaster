# MechaCore 库逐模块移植分析

本文档逐一分析 `core/` 目录下的每个库模块，评估其对飞镖项目的**实用价值**和**移植难度**。

> [!TIP]
> 📗 推荐移植 · 📙 可选移植 · 📕 不推荐移植（与飞镖无关或成本过高）

---

## 一、Alg 层（纯算法，零硬件依赖）

### 📗 Alg/PID — PID 控制器
- **依赖**：无外部依赖，纯数学运算
- **价值**：`float` 精度、积分分离、前馈接口，比现有 `int` 精度的 `PID.cpp` 更加现代和稳健
- **移植难度**：⭐ (极低)，直接 `#include` 即可替换
- **建议**：**优先移植**，替换现有 `Middle_Plugin/PID`

### 📗 Alg/Filter — 滤波器集合
- **依赖**：无外部依赖
- **内容**：`KalmanFilter` + `TDFilter` + `LPFFilter` + `LMFFilter`
- **价值**：项目中已有一份 `Filter.cpp/hpp`，但 `core` 版更完善（有 Getter、可重初始化）
- **移植难度**：⭐ (极低)
- **建议**：**推荐替换**现有版本，接口更清晰

### 📗 Alg/ADRC — 自抗扰控制器
- **依赖**：仅依赖自身 `AdrcBase.hpp`（无硬件依赖）
- **内容**：一阶 `FirstLADRC` + 二阶 `SecondLADRC`
- **价值**：适合 Yaw 轴等对抗扰性能要求高的场景，比 PID 更好的抗扰特性
- **移植难度**：⭐ (极低)
- **建议**：**推荐引入**作为 PID 的高级替代方案

### 📗 Alg/UtilityFunction/SlopePlanning — 斜坡规划
- **依赖**：无外部依赖
- **价值**：平滑控制输出变化（限制加速度），可用于升降电机等场景
- **移植难度**：⭐ (极低)
- **建议**：**推荐移植**，实用性强

### 📙 Alg/FSM — 有限状态机
- **依赖**：仅 `stdint.h`
- **内容**：通用 FSM 框架，带时间计数的状态管理
- **价值**：飞镖的 `AutoLaunch` 状态机已经是自定义实现。此库较为**简单通用**，可用来替代或规范化现有状态机逻辑
- **移植难度**：⭐⭐ (低)，需重构 AutoLaunch 状态机以适配
- **建议**：**可选**，如果你想统一状态机范式可以引入

### 📙 Alg/Feedforward — 前馈控制器
- **依赖**：无外部依赖
- **内容**：
  - `Acceleration` / `Velocity` / `Gravity` / `Friction`：通用前馈组件 📗
  - `GimbalFullCompensation`：云台专用摩擦+惯量补偿 📗（适合 Yaw 轴）
  - `UDE`：扰动观测器 📙
  - `Uphill`：上坡前馈（系数矩阵需 MATLAB 生成）📕
- **移植难度**：⭐ (极低)，纯算法类
- **建议**：`Gravity` + `GimbalFullCompensation` 对飞镖 Yaw 轴有实用价值，**建议按需选用**。`Uphill` 是底盘坡度补偿，飞镖不需要

### 📕 Alg/ChassisCalculation — 底盘运动学
- **内容**：全向轮正/逆运动学 (`Omni_FK`, `Omni_IK`, `Omni_ID`)、舵轮 (`StringWheel`)
- **价值**：**飞镖没有底盘**，完全不适用
- **建议**：**不移植**

### 📕 Alg/VMC — 虚拟模型控制
- **内容**：轮腿式底盘虚拟力仿真
- **价值**：**飞镖不使用轮腿结构**
- **建议**：**不移植**

### 📕 Alg/PowerControl — 功率控制
- **内容**：底盘功率限制算法
- **价值**：**飞镖无底盘功率问题**
- **建议**：**不移植** (PowerControl-TestVersion 同理)

### 📕 Alg/UtilityFunction/drv_math.h — 数学工具
- **内容**：一些宏定义（限幅、绝对值等）
- **价值**：项目中已有类似的 `MyTool` 工具类
- **建议**：**不移植**，避免重复

## 二、BSP 层（传感器/设备驱动，有硬件依赖）

### 📗 BSP/RemoteControl/DT7 — 遥控器驱动
- **依赖**：需要 `StateWatch`（掉线检测）
- **内容**：DT7 遥控器数据解析，包含摇杆、拨杆、鼠标和键盘
- **价值**：项目中已有一份 `Middle_Plugin/DT7.cpp`，`core` 版**集成了 StateWatch 掉线检测**，比现有版本更安全
- **移植难度**：⭐⭐ (低)，需同时引入 `StateWatch`
- **建议**：**推荐移植**，体验更现代的接口

### 📙 BSP/Motor/MotorBase — 电机驱动基类
- **依赖**：`StateWatch` + `BuzzerManager` + `HAL::CAN`（深度耦合）
- **内容**：模板类 `MotorBase<N>`，带国际单位转换、掉线检测、蜂鸣器报警
- **价值**：设计精良，但依赖链太深（需同时移植 CAN HAL + StateWatch + BuzzerManager）
- **移植难度**：⭐⭐⭐⭐ (高)
- **建议**：**暂缓**。如果做全量框架迁移再考虑

### 📙 BSP/IMU/HI12 — HI12陀螺仪驱动
- **依赖**：`HI12Base.hpp`（含帧校验、时间戳）
- **内容**：解析 HI12 串口 IMU 数据（加速度、角速度、欧拉角、四元数）
- **价值**：**飞镖目前不使用 IMU**。但如果未来增加 IMU 可直接使用
- **移植难度**：⭐⭐ (低)
- **建议**：**按需移植**，目前不急

### 📕 BSP/RMRefereeSystem — 裁判系统驱动
- **价值**：项目中**已有独立的** `RMRefereeSystem/` 实现，且已适配飞镖协议
- **建议**：**不移植**，已有等效实现

## 三、HAL 层（硬件抽象，深度耦合寄存器）

### 📙 HAL/DWT — 高精度计时器
- **依赖**：ARM Cortex-M DWT 寄存器 + `<atomic>` + `<chrono>`
- **内容**：单例模式、微秒/毫秒级计时、精确延时
- **价值**：比 `HAL_GetTick()` 更精确（周期级），适合高频 PID 的时间基准
- **移植难度**：⭐⭐ (低)，直接可用于 Cortex-M4
- **建议**：**可选移植**。当需要微秒级时间戳（如变步长PID）时有用

### 📕 HAL/CAN — CAN总线抽象
- **依赖**：`can_bus.hpp` + `can_device.hpp`（需改造 CubeMX CAN 初始化流程）
- **内容**：面向对象的 CAN 收发框架
- **价值**：设计优雅，但改造成本太高——需要重写整个 CAN 回调和初始化流程
- **移植难度**：⭐⭐⭐⭐⭐ (极高)
- **建议**：**不推荐单独移植**。仅在全量框架迁移时考虑

### 📕 HAL/LOGGER — 日志系统
- **依赖**：串口 HAL
- **建议**：**不移植**，飞镖已有 Vofa 调试通道

### 📕 HAL/PWM — PWM 封装
- **建议**：**不移植**，飞镖 PWM 使用简单

### 📕 HAL/ASSERT — 断言
- **建议**：**不移植**，调试场景有限

### 📕 APP/PowerLimit — 功率限制任务

- **建议**：**不移植**，飞镖不适用

### 📕 APP/Heat_Detector — 热量控制
- **内容**：枪口射速/热量管理
- **建议**：**不移植**，飞镖不适用

### 📕 APP/Referee — 裁判系统应用
- **建议**：**不移植**，已有独立实现

### 📕 APP/control.cpp — 控制逻辑示例
- **建议**：**不移植**，仅参考代码
