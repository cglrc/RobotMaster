# 电机掉线检测和蜂鸣器报警功能说明

## 功能概述

本功能实现了对8个电机的掉线检测和蜂鸣器报警功能。当某个电机超过100ms没有接收到数据时，系统会通过蜂鸣器发出报警声。

## 电机ID映射

| ID | CAN ID | 电机名称 | 闪烁/报警次数 |
| :--- | :--- | :--- | :--- |
| 1 | 0x201 | 预留（暂未使用） | 1次 |
| 2 | 0x202 | 左侧升降 M2006 | 2次 |
| 3 | 0x203 | 右侧上膛 M3508 | 3次 |
| 4 | 0x204 | 左侧上膛 M3508 | 4次 |
| 5 | 0x205 | Yaw轴 M6020 | 5次 |
| 6 | 0x206 | 右侧升降 M2006 | 6次 |
| 7 | 0x207 | 拉簧/滑块 M3508 | 7次 |
| 8 | 0x208 | 角度传感器 M3508 | 8次 |

## 报警规则

1. **掉线检测**：电机超过100ms没有接收到CAN数据时，判定为掉线
2. **报警次数**：每个电机的报警次数与其 **ID 编号一致**（例如：3号电机响3次）
3. **报警时长**：每次报警持续50ms
4. **报警间隔**：每次报警之间间隔50ms
5. **多电机处理**：当有多个电机同时掉线时，按顺序依次报警
6. **电机间等待**：完成一个电机的报警后等待750ms，然后开始下一个电机的报警
7. **循环等待**：完成所有电机的报警后等待750ms，然后重新开始报警
8. **恢复检测**：当电机重新接收到数据时，立即停止报警并从队列中移除

## 时间参数

- **掉线阈值**：100ms
- **报警时长**：50ms (BUZZER_ON_TIME_MS)
- **报警间隔**：50ms (BUZZER_OFF_TIME_MS)
- **电机间等待**：750ms (BUZZER_CYCLE_WAIT_MS)
- **循环等待**：750ms
- **检测频率**：每10ms检测一次
- **蜂鸣器频率**：523Hz（中音1）

## 蜂鸣器控制

- **频率**：523Hz（中音1）
- **占空比**：20%
- **优先级**：电机掉线报警优先于音乐播放

## 使用方法

1. **初始化**：系统启动时自动调用 `MotorOfflineDetector_Init()`
2. **状态更新**：在接收到电机 CAN 数据时调用 `MotorOfflineDetector_UpdateMotorStatus()`
3. **定期检测**：在后台任务或定时器中定期调用 `MotorOfflineDetector_Update()`

## 注意事项

1. 该功能使用 TIM12 的 CH1 通道控制蜂鸣器。
2. 当有电机掉线报警时，音乐播放功能（`playState`）会被置为 0（暂停）。
3. 多个电机同时掉线时，按电机 ID 递增顺序依次报警。
4. 电机恢复连接后，报警会立即停止并从队列中移除。
5. 指示灯逻辑：掉线电机的对应 LED (PG1~PG8) 会闪烁 ID 次，然后休眠 1.5s 循环。

## 多电机报警示例

假设 3 号和 5 号电机同时掉线：

1. **第一轮报警**：
   - 先响 3 号电机：响 **3** 次（每次 50ms，间隔 50ms）。
   - 等待 750ms。
   - 再响 5 号电机：响 **5** 次（每次 50ms，间隔 50ms）。
   - 等待 750ms。

2. **第二轮报警**：
   - 重复上述过程，直到电机恢复。

**注意**：每个电机在每轮循环中只会响一次自己的完整次数。

## 调试信息

可以通过以下方式监控电机状态：

```cpp
// 检查电机是否掉线
if (motor_status[motor_id].is_offline) {
    // 电机已掉线
}

// 检查蜂鸣器状态
if (buzzer_control.state != BUZZER_IDLE) {
    // 蜂鸣器正在报警
}

// 检查队列中的电机数量
uint8_t queue_count = buzzer_control.queue_count;
```
